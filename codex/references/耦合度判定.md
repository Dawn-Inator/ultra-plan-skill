# 模块耦合度判定

> Phase 2 主 agent 拆解模块清单后，必须为每个模块标耦合度。这决定了 Phase 3 写文档时**主 agent 串行写** vs **派 subagent 并行写**的策略。

---

## 判定标准

按以下条件**逐项检查**，命中任一"高耦合"条件 → 高耦合。

| 条件 | 高耦合 | 中耦合 | 低耦合 |
|---|---|---|---|
| 模块间共享 ORM 表 / DB schema / GraphQL type | ✅ | - | - |
| 模块间共享 类型定义（Pydantic / TS interface / proto）| ✅ | - | - |
| 模块间互调 API（A 模块的 endpoint 调用 B 模块的 endpoint）| ✅ | - | - |
| 模块间共享前端 hook / store / context | - | ✅ | - |
| 模块间共享非通用 UI 组件 | - | ✅ | - |
| 模块间共享业务术语（同一术语指代不同含义会出 bug）| - | ✅ | - |
| 模块共享配置 key / env 变量 | - | ✅ | - |
| 模块完全独立（自己的表 / 自己的端点 / 自己的页面）| - | - | ✅ |
| 模块仅在 00_总体规划.md 索引页出现 | - | - | ✅ |
| 模块只调用框架级公共函数（如 logger、auth、http client）| - | - | ✅ |

---

## 写文档策略

### 全部主 agent 串行写（保守）

满足任一条件即采纳：

- 模块清单里 ≥ 30% 模块标"高耦合"
- 模块总数 ≤ 6（数量少，串行成本低）
- 用户在 Phase 0 选了"深度调研"（说明需求复杂，并行风险高）

### 混合模式（推荐）

- 主 agent 串行写：00_总体规划.md + 高耦合模块 md + 中耦合模块 md
- subagent 并行写：低耦合模块 md（每个 subagent 1-2 个 md）

### 全部并行（激进，慎用）

仅当**全部模块都是低耦合**时采纳。罕见。

---

## 派发并行 subagent 时的额外约束

即使是低耦合模块，并行派 subagent 写时仍要：

1. **00_总体规划.md 必须先写完**（作为 single source of truth）
2. **每个 subagent prompt 必须包含 00_总体规划.md 全文路径**并要求复述
3. **subagent 写完后主 agent 必须扫一遍**，检查：
   - 术语一致性（同一概念是否多个名字）
   - 路径一致性（输出目录是否拼对）
   - 模板遵循度（是否都有 8 节）
   - 派发前红线段（第 8 节）是否一致

## 决策示例

### 示例 1：管理后台整套（典型高耦合）

模块：账户认证 / 内容审核 / 财务 / 隐私 / 用户管理 / 角色权限 / 通知 / 设置

判定：
- 角色权限模块定义 RBAC，被全部其他模块引用 → 高耦合
- 财务 / 隐私 / 通知共享审计日志表 → 高耦合
- 70%+ 模块标高耦合

→ **全部主 agent 串行写**。

### 示例 2：用户个人主页 v1（典型低耦合）

模块：基本信息编辑 / 头像上传 / 简介编辑 / 隐私设置开关 / 注销账号

判定：
- 5 个子功能都是 user 表的不同字段更新
- 各自独立 endpoint，互不调用
- 全部低耦合

→ 主 agent 写 00_总体规划.md，**派 3 个 subagent 并行写 5 个模块 md**（每 agent 1-2 个）。

### 示例 3：支付系统 v1（典型混合）

模块：支付意图创建 / 支付方式选择 / 支付回调处理 / 退款 / 对账 / 收据导出 / 风控

判定：
- 支付意图创建 / 回调 / 退款 共享 PaymentIntent schema → 高耦合
- 对账 / 风控 共享支付记录表读 → 中耦合
- 收据导出独立 endpoint，独立 PDF 生成 → 低耦合

→ **混合**：主 agent 串行写 5 个高/中耦合 md，派 1 个 subagent 写收据导出 md。

---

## 反模式（避免）

- ❌ "为了快，把高耦合模块也派 subagent 并行写" → 必然术语 / schema 不一致
- ❌ "为了稳，所有需求都串行写" → 大需求会让主 session 上下文爆
- ❌ "判定不清就默认低耦合" → 默认应是高耦合（保守优先）
- ❌ "判定一次后不再调整" → 写文档过程中发现耦合度估错了，应立刻停下重判
